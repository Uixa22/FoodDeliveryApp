# FoodDeliveryApp
FoodDeliveryApp — современное микросервисное приложение доставки еды и продуктов, аналог Яндекс.Еды, Delivery Club и Самоката.
Технологический стек

Java 21
Spring Boot 3.2+
Spring Data JPA + Hibernate
PostgreSQL
Apache Kafka (event-driven коммуникация)
Docker + docker-compose (вся инфраструктура)
Lombok, MapStruct, Springdoc OpenAPI (Swagger UI)
Gradle (мультимодульный проект)

Архитектура
Проект состоит из трёх независимых микросервисов:

order-service — центральный сервис: создание заказа, управление статусами, оркестрация процессов
payment-service — обработка платежей (интеграция с order по HTTP REST API)
delivery-service — назначение курьеров и отслеживание доставки (асинхронное взаимодействие через Kafka)

Основной функционал

Создание заказа пользователем
Проведение платежа (успех / отказ → смена статуса заказа)
Автоматическое назначение ближайшего курьера
Обновление статусов в реальном времени:
PENDING_PAYMENT → PAID / PAYMENT_FAILED
PAID → AWAITING_COURIER → IN_DELIVERY → DELIVERED / CANCELLED

Асинхронная обработка событий через Kafka (например, "OrderPaid" → триггер назначения курьера)
Полная инфраструктура (PostgreSQL + Kafka + Zookeeper) поднимается одной командой docker-compose up

Особенности реализации

Event-driven архитектура: order-service публикует события в Kafka → payment и delivery подписываются независимо
Декларативные HTTP-клиенты (@HttpExchange) для общения order ↔ payment
Обработка состояний заказа через конечные автоматы (FSM-подход)
Swagger UI для тестирования API каждого сервиса
Тестовые данные и миграции через Flyway / Liquibase 
Полная изоляция сервисов: каждый имеет свою БД-схему 

Проект демонстрирует современные практики микросервисов: асинхронность, отказоустойчивость, масштабируемость и чистую архитектуру.
Готов к расширению: добавление аутентификации (Keycloak), мониторинга (Prometheus + Grafana), CI/CD (GitHub Actions), фронтенда (React/Vue).
Репозиторий: мультимодульный Gradle-проект с чёткой структурой и документацией.
